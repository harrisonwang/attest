# attest — 摸底实验记录

> 日期：2026-07-09 · 脚本：[probe/drift-probe.py](probe/drift-probe.py)（原样归档，标注了已知缺陷）
> 结论：这条路走得通。5 份真实的 agent 文档，约 230 个 token，绑定率 60% 到 90%，按"只报确定对不上的、其余不出声"的策略，零误报。每一个没覆盖到的缺口都对到一个能机械实现的检查逻辑，没有一个需要靠理解语义来补。

## 为什么做这个实验

立项讨论的时候，最核心的质疑是这样一句话：CLAUDE.md、AGENTS.md、SKILL.md 没有强格式规范——你抽不出来就没法核验。

探针要回答的问题不是"能不能理解文档"，而是另一个更实际的问题：不理解文档、只做 CommonMark 提取加上仓库配对，在真实文件上绑定率能到多少、误报率能到多低？120 行一次性 Python，大概 90 分钟写完，全程没用任何 LLM。

## 怎么做的

第一步把反引号包住的内容全抽出来，围栏代码块（bash、sh、没标语言的都算）里逐行取命令。

第二步对每个 token 挨个尝试配对。路径存在吗？在 package.json 的 scripts 或者 Makefile 里吗？是 workspace 的包名吗？系统 PATH 上有这个命令吗？全大写的环境变量能在源码里搜到吗？像标识符的词能在源码里搜到吗？

第三步判结果。对上了就计数。形态完备但对不上的就报 BROKEN。其他的都沉默，不报。

## 数据

测了五份文件。vouch 的 CLAUDE.md 抽了 15 个 token，9 个直接对上，6 个没对上但全是行内多词命令（像 `cargo test`、`git commit`），因为探针只对围栏块做了命令解析——补上就能对，真实绑定率大约 14/15。git-why 的 CLAUDE.md 抽了 22 个，18 个对上，剩下有 `ai` 这种歧义词，正确沉默。kortiv 的 AGENTS.md 抽了 16 个，14 个对上，`UI` 和 `.env` 正确沉默——.env 本来就不应该在仓库里。

jadeenvoy 的 AGENTS.md 最大，抽了 148 个，68 个对上，6 个报 BROKEN，12 个跳过，62 个未绑定。所有的都可以归因到缺少对应的检查模块。spoor 的 SKILL.md 抽了 27 个，13 个对上，两个 BROKEN 事后复核都是探针自己的缺陷——基目录规则不对、没有配置键检查。没对上的多来自没标语言围栏块里的 ASCII 大纲。

## 六个教训，六个设计决策

行内多词命令没绑定——`cargo test`、`git commit` 这种，因为探针只对围栏块做了命令解析。解决方法是行内 span 和围栏块走同一套命令解析器。

SKILL.md 里的 `phases/` 报了 BROKEN，但实际上相对 SKILL.md 自己的位置是存在的，五个文件全在。因为探针只从仓库根开始找。解决方法是四级基目录，文档自身所在目录最优先。

"Phase 1: Discovery..." 这种 ASCII 流程大纲混进来了，因为没标语言的围栏块被当成了 shell。解决方法是只解析显式标了 bash、sh、shell、zsh、console 的围栏块。

Go 的 import 路径 `net/http`、`mattn/go-sqlite3` 被当成文件路径报了 BROKEN。解决方法是在 go.mod 和标准库里查，提供了单独的 Go import 检查模块。

`.scaffold-docs.yml` 里真实存在的键 `ask_audience` 报了 BROKEN，因为没有配置键这个检查类别。解决方法是加一个配置键检查模块。

jadeenvoy 的 CONVENTIONS.md 报了 BROKEN，但原文写的是"未配置；可建 symlink"——文档自己声明了这个东西不存在。解决方法是让每个 finding 带上所在行的原文，加语境守卫，否定语境的 broken 降成 suspect。更严苛的语境判断交给 claims.lock。

另外有个正向发现。API 路由（像 `/v1/runtimes`）和 ID 前缀字面量（像 `agt-`、`sess-`）目前沉默，但都可以被机械的检查模块覆盖——路由注册表 grep 和字符串字面量 grep——归到远期生态清单里。

## 证明了什么，没证明什么

验证了的事情：反引号约定在真实的 agent 文档里确实足够普遍，CommonMark 提取就能拿到高密度的可配对 token。仓库当裁判的这种方式精度极高，所有假的 BROKEN 都来自探针实现本身的缺陷——缺检查模块、缺基目录规则，不是方法有问题。误差不对称成立——没对上的都安静沉默了，没有一例因为"理解错了句子"而误报。

还没验证的事情——这是留到第一阶段批量测的。金档对历史真实 drift 的召回率（探针测的是当前快照的绑定率，不是对已知漂移的检出），那个 60% 的止损条件就悬在这里。反引号之外的散文引用到底占多少比例、形状候选实际精度怎么样。大型多语言 monorepo 上的性能和路径查找规则表现如何。
